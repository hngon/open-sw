// 0317 수업
서로 다른 사용자가 같은 시기에 병합할 때 충돌이 일어나면 3way merge 방식으로 구분
에를 들어 version 1, 2, 3, 4 중 사용자1이 3에서 수정 후 병합, 사용자2가 4에서 수정 후 병합하면(같은 시기에 병합한다 가정.) 
base code는 version3에서 끌어와 수정한 사용자1이 기준임.
해결하려면 사용자2가 base code를 사용자1이 병합한 코드를 최신화 후 서로 상의, 코드 최신화 하고 push하기(pull request 활용)
그럼 사용자1이 병합한 코드에 사용자2의 코드가 합쳐짐.

오픈소스애서는 원본 저장소를 복사한 복제 저장소(나만의 저장소)에 push하고 원본 저장소 주인에게 자신의 저장소인 복제 저장소에서 pull 요구(fork 방식), 
원본 저장소 주인이 확인 후 merge해주면 성공

// 실습 내용 
깃헙 저장소 url을 협업자에게 전송, 협업자는 수정하려면 fork해서 나만의 저장소를 만듦(만약 원본 저장소가 코드가 바뀌면 synkfork해서 최신화),
협업자가 commit 후 Pull requests하면 원본 저장소에게 변경사항이 전송됨.
관리자가 Pull requests 확인 후 merge하면 반영이 됨.
